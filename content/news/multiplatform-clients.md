# OpenClaw 多平台客户端详解

OpenClaw 的设计理念是"任何操作系统、任何平台都可以使用"。本文将详细介绍各平台客户端的功能和使用方式。

## 支持的平台

### macOS 客户端

macOS 客户端提供两种形式：

**菜单栏应用**
- **菜单栏常驻** - 随时从屏幕顶部快速访问
- **轻量级** - 不占用大量系统资源
- **通知集成** - 支持系统通知
- **快捷键支持** - 可配置全局快捷键唤醒

**macOS 伴侣应用（Beta）**
- 专为 macOS 15+ 设计
- 菜单栏快速访问
- 与系统深度集成
- 支持 Apple Silicon 优化

### iOS 客户端

原生 iOS 应用，支持 iPhone 和 iPad：

- **原生体验** - 遵循 Apple 人机交互指南
- **推送通知** - 不打开应用也能收到消息
- **Siri 集成** - 可通过 Siri 快速启动
- **跨设备同步** - 与其他 Apple 设备同步状态

### Android 客户端

原生 Android 应用：

- ** Material Design** - 遵循 Material Design 3 设计语言
- **后台运行** - 支持持续后台运行
- **语音输入** - 支持连续语音对话
- **小部件支持** - 可添加桌面小部件快速访问

## 客户端架构

OpenClaw 采用 Gateway + Nodes 的分布式架构：

```
┌─────────────────────────────────────┐
│            Gateway                   │
│  (运行在专用主机上，作为中央处理单元)  │
└──────────────┬──────────────────────┘
               │ WebSocket
       ┌───────┴───────┐
       │               │
  ┌────▼────┐    ┌─────▼────┐
  │ macOS   │    │  iOS     │
  │ Menu    │    │  App     │
  └─────────┘    └──────────┘
       │               │
  ┌────▼──────────────▼────┐
  │      Android App        │
  └─────────────────────────┘
```

### Gateway 的角色

- 维护所有渠道连接
- 处理消息路由
- 管理会话状态
- 提供统一 API

### Nodes 的角色

- 提供设备原生能力
- 麦克风、摄像头、GPS 等
- 本地通知
- 离线消息缓存

## 跨设备同步

### 会话同步

OpenClaw 支持跨设备同步会话：

- 在手机上开始的对话，可以在电脑上继续
- 历史消息自动同步
- 上下文完整保留

### 配置同步

- 渠道配置自动同步
- 技能设置保持一致
- 个人偏好跨设备统一

### 实时状态

- 设备在线状态实时显示
- 消息已读状态同步
- 输入状态跨设备可见

## 使用场景

### 场景一：手机接收、电脑处理

1. 在手机上通过 WhatsApp 收到工作消息
2. 在电脑上打开 Control UI 进行处理
3. 处理结果直接通过手机渠道回复

### 场景二：语音输入、文字输出

1. 在 Android 上用语音询问复杂问题
2. 在电脑上查看详细的文字分析报告
3. 通过飞书渠道转发给团队

### 场景三：随时随地查询

- 驾车时用语音查询导航
- 走路时用手机查看待办事项
- 坐在电脑前用完整功能深度工作

## 安装与配置

### 获取客户端

客户端可以通过以下方式获取：

- 官方网站下载
- App Store / Google Play
- GitHub Releases

### 首次配置

1. 在运行 Gateway 的机器上完成 onboarding
2. 在客户端登录相同账号
3. 自动发现并连接 Gateway

### 故障排除

如果客户端无法连接：

1. 检查 Gateway 是否运行：`openclaw status`
2. 检查网络连接
3. 确认认证配置正确
4. 查看日志获取详细错误信息

## 安全考虑

### 本地认证

- 支持生物识别（Face ID、指纹）
- 支持设备密码
- token 本地加密存储

### 传输安全

- WebSocket 全程加密
- Tailscale 隧道安全传输
- 可选端到端加密

### 设备管理

- 可在 Control UI 中查看已连接设备
- 支持远程登出设备
- 设备授权需手动确认

## 与其他功能的结合

### 与渠道结合

多平台客户端可以接收所有已配置渠道的消息：

- WhatsApp 消息推送到手机
- Telegram 消息在电脑上处理
- 飞书消息统一管理

### 与语音结合

- iOS 客户端支持 Siri 唤醒
- Android 客户端支持持续语音对话
- macOS 客户端支持语音输入

### 与 Canvas 结合

在支持可视化功能的客户端上：

- 查看 AI 生成的数据图表
- 直接在 Canvas 上交互操作
- 实时查看动态内容更新
