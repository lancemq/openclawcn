# 语音功能最佳实践

OpenClaw 的语音功能让它成为了一个真正"随时在线"的助手。本文将分享如何在各种场景中高效使用语音功能。

## 语音功能的价值

语音交互的核心价值在于：

- **释放双手** - 双手忙碌时也能与 AI 交互
- **快速输入** - 说话比打字更快
- **随时随地** - 不需要打开特定应用
- **自然交互** - 像与人对话一样自然

## 推荐的语音使用场景

### 1. 驾驶场景

驾驶时是最需要语音交互的场景：

- "导航到最近的加油站"
- "给老婆打电话"
- "播放周杰伦的歌"
- "提醒我二十分钟后打电话"

### 2. 厨房场景

烹饪时双手沾满食材：

- "番茄炒蛋的做法"
- "设置十分钟的计时器"
- "帮我算一下 200 克面粉是多少杯"
- "列一个采购清单"

### 3. 运动健身场景

运动中不方便操作手机：

- "记录今天跑步 5 公里"
- "给我一个腹肌训练计划"
- "现在播放适合跑步的音乐"
- "消耗了多少卡路里"

### 4. 日常助手场景

日常生活助手：

- "今天天气怎么样"
- "提醒我明天早上九点开会"
- "查一下从深圳到广州的高铁"
- "帮我记一下老婆的生日礼物idea"

## 语音指令技巧

### 结构化表达

使用结构化的表达方式让识别更准确：

```
[动作] + [对象] + [细节]
```

例如：
- "帮我查询深圳天气"
- "设置下午三点的会议提醒"
- "播放周杰伦的晴天"

### 连续对话

OpenClaw 支持连续语音对话：

1. 唤醒助手
2. 提出第一个问题
3. 助手回答后可继续追问
4. 不需要每次都重新唤醒

### 明确上下文

在连续对话中明确上下文：

- "昨天让你查的股票现在多少钱"
- "继续刚才的食谱，下一步是什么"

## 常见问题解决

### 语音识别不准

- 尽量在安静环境使用
- 发音清晰，语速适中
- 靠近设备麦克风
- 检查网络连接

### 响应太慢

- 确认网络稳定
- 检查 Gateway 是否正常运行
- 考虑使用本地模型（如果支持）

### 无法唤醒

- 检查是否启用了语音唤醒功能
- 确认麦克风权限已授予
- 查看设备是否支持语音唤醒

## 与其他功能结合

### 语音 + 渠道

通过语音发送消息到各个渠道：

- "给张三发微信说..."
- "在飞书上通知团队..."
- "发邮件给老板..."

### 语音 + 技能

调用特定技能：

- "用待办技能记录..."
- "让翻译技能帮我翻一下..."
- "用计算技能算一下..."

### 语音 + Canvas

语音与 Canvas 结合：

- "展示最近的销售数据"
- "把结果做成图表"
- "在 Canvas 上显示"

## 安全注意事项

### 隐私保护

- 语音数据默认本地处理
- 可在设置中查看数据处理方式
- 敏感场景可关闭语音功能

### 安全访问

- 设置语音解锁密码
- 在公共场合使用耳机
- 定期检查语音访问记录

## 配置建议

### 推荐配置

1. 启用语音唤醒
2. 设置识别语言为中文
3. 启用语音反馈（听到回复）
4. 配置常用快捷指令

### 根据场景调整

- 驾驶模式：最大音量，启用播报
- 安静环境：关闭播报，更精准识别
- 会议场景：使用耳机，避免外放

## 进阶使用

### 自定义唤醒词

可以自定义唤醒词，但要注意：

- 避免常用词导致误触发
- 发音要清晰明确
- 建议 3-4 个字

### 快捷指令

创建常用指令的快捷方式：

- "上班" = "查看今天日程 + 天气 + 交通"
- "会议" = "进入专注模式 + 设置提醒"
- "回家" = "播放舒缓音乐 + 打开空调"

## 小结

语音功能是 OpenClaw 的一大亮点，它让 AI 助手真正融入了日常生活和工作。掌握这些技巧，可以大幅提升使用效率。
