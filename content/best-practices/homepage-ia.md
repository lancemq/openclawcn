# 首次接触 OpenClaw 的阅读顺序

第一次接触 OpenClaw 时，最常见的问题不是资料太少，而是不知道先看什么。

## 推荐顺序

1. 先看产品定位
2. 再看核心能力与适用场景
3. 然后进入安装、onboarding 和 Control UI
4. 最后再看 channels、hooks、安全和更新跟踪

## 为什么这样更轻松

- 不会一开始就被复杂配置劝退
- 能先建立整体地图，再进入细节
- 遇到问题时更容易知道该去文档、FAQ 还是社区

## 最不推荐的顺序

- 直接先看高级配置
- 还没理解结构就去接多个渠道
- 一开始就把安全、hooks、网关和运维混在一起读
