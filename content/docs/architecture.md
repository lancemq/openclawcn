# 架构约束

为了让项目从一开始就清晰可维护，本项目默认遵循以下约束：

## 内容优先

- 首页优先回答“OpenClaw 是什么”
- 文档优先回答“如何理解和开始使用”
- 新闻优先承接项目更新与社区动态

## 中文用户优先

- 术语尽量用中文解释
- 安装与使用路径按中文用户习惯组织
- 社区、反馈和 GitHub 入口保持清晰可见

## 技术边界清晰

- 首页、新闻和文档优先由内容驱动
- 第 1 步不急于接入复杂后台
- 第 2 步再扩展搜索、反馈和动态接口
- 部署平台属于内部实现，不作为对外内容主题
