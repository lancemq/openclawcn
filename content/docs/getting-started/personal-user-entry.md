# 如果你是个人用户，应该先走哪条入口

个人用户最容易遇到的问题，不是资料太少，而是入口太多。

常见状态通常是：

- 想先把 OpenClaw 变成自己的长期助手
- 想接一个自己常用的聊天入口
- 也想试模型、试技能、试语音

但如果一开始把这些都混在一起，成本和排错难度会一起上来。

## 个人用户最值得先确认的 3 件事

### 1. 你要的是“长期助手”，还是“临时玩一玩”

如果你只是想先感受一下 OpenClaw，先跑通本地最小链路就够。

如果你是准备长期使用，才值得继续看：

- 渠道接入
- 远程访问
- 记忆和会话边界

### 2. 你更习惯哪种入口

个人用户通常会在三种入口之间犹豫：

- Dashboard：更适合看状态和管理面
- WebChat：更适合先测试聊天和上下文
- Telegram / WhatsApp / 飞书：更适合进入真实消息流

更稳的顺序通常是：

1. 先确认 Dashboard 正常
2. 再用 WebChat 验证最小聊天链路
3. 最后再接一个最常用的聊天渠道

### 3. 你是否真的需要一开始就上 Skills

很多个人用户上手时最容易冲动扩能力。

但更好的节奏通常是：

- 先把基础聊天和会话跑稳
- 再确定真正缺的是哪类能力
- 最后只接最必要的 Skills 或 Tools

## 个人用户最常见的误区

### 一开始就接多个渠道

这样最容易把入口问题、会话问题和模型问题混在一起。

### 把模型调优放在入口之前

如果你连最常用入口都还没稳定跑通，过早折腾模型组合通常收益不大。

### 把 Skills 当成“越早越好”

对个人用户来说，很多时候稳定入口和长期使用节奏比技能数量更重要。

## 一条最适合个人用户的默认路线

1. 先本地跑通
2. 再选一个真实入口
3. 再补模型和 Skills
4. 最后再考虑远程访问、语音或多设备协同
