# 模型、配置和安全这三页分别什么时候看

站里最容易被一起点开的三页通常是：

- [模型选择指南](/models)
- [关键配置](/configurations)
- [安全实践](/security)

它们都重要，但解决的问题不一样。

## 什么时候看模型页

如果你正在问这些问题，就先去模型页：

- 默认模型该先选什么
- Claude / GPT / Gemini / DeepSeek 怎么取舍
- 成本怎么控制
- 什么时候需要 fallback

一句话：  
只要你的问题核心是“选哪种模型更合适”，就先去 [模型选择指南](/models)。

## 什么时候看配置页

如果你正在问这些问题，就先去配置页：

- 配置文件先改哪里
- Gateway、channels、session、skills 这些层怎么分
- `openclaw.json` 里哪些字段最关键
- 我该用向导、CLI 还是直接改文件

一句话：  
只要你的问题核心是“怎么把系统配对”，就先去 [关键配置](/configurations)。

## 什么时候看安全页

如果你正在问这些问题，就先去安全页：

- 入口是否暴露得太宽
- 远程访问怎么做更稳
- trusted proxy 和 Tailscale / SSH 怎么选
- 哪些动作应该保留人工审批

一句话：  
只要你的问题核心是“这件事安不安全、边界对不对”，就先去 [安全实践](/security)。

## 最常见的误区

### 把模型问题带去配置页

例如：

- “Claude 和 Gemini 该选哪个”
- “预算有限该怎么搭配”

这类先看模型页更有效。

### 把安全问题带去配置页

例如：

- “能不能直接暴露公网”
- “trustedProxies 应该怎么设”

这类先看安全页。

### 把配置问题带去模型页

例如：

- “默认模型写在哪层”
- “fallback 放哪个字段”

这类先看配置页。

## 一张最短判断表

- 选模型：去 [模型选择指南](/models)
- 配系统：去 [关键配置](/configurations)
- 看边界：去 [安全实践](/security)

## 2026 年 3 月 24 日的中文阅读误区

近期公开可访问的中文教程站和社区文章，在模型、配置和安全这三类内容上，经常会出现一种很自然但很容易带偏人的写法：

- 先讲模型推荐
- 再给一份配置示例
- 最后顺手提一句安全

这对快速建立印象很有帮助，但如果你已经要真正落地，阅读顺序更稳的方式通常还是：

1. 先看配置结构，确认系统是怎么组织的  
2. 再看模型选择，决定默认 provider 和 fallback  
3. 最后看安全边界，收入口、反向代理和远程访问

也就是说：

- 中文外部资料更适合帮你建立“方向感”
- 站内这三页更适合帮你把“真正的判断顺序”拉回来

## 如果你三个问题都混在一起

建议顺序是：

1. 先看 [关键配置](/configurations)，确定结构
2. 再看 [模型选择指南](/models)，选默认模型
3. 最后看 [安全实践](/security)，收入口和边界

这样比反过来看更不容易混乱。
