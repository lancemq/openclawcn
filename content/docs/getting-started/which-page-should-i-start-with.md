# 我应该先从哪个页面开始

第一次进入站点时，最容易浪费时间的地方不是内容不够，而是入口太多。  
这页不解释具体功能，只回答一个问题：你现在最应该先点哪里。

## 如果你完全是第一次接触

优先从下面两页开始：

- [OpenClaw 是什么](/docs/getting-started/what-is-openclaw)
- [快速入门](/docs/getting-started/getting-started)

适合：

- 还不知道 OpenClaw 适不适合自己
- 还没安装
- 只想先跑通一条最小链路

## 如果你知道自己要“按顺序学习”

优先进入：

- [学习路径](/paths)

适合：

- 想按新手、渠道接入、远程运行、团队运维来走
- 不想自己在文档和视频之间来回跳
- 希望先有一条连续路线再深入

## 如果你已经知道自己关心的主题

优先进入：

- [主题中心](/topics)

适合：

- 已经知道自己关心安装、Gateway、模型、安全、渠道或 Skills
- 不需要完整主线，只想围绕当前问题看同主题资料

## 如果你更习惯先看演示

优先进入：

- [视频教程](/videos)

适合：

- 想先看别人怎么操作
- 还不想先读长文档
- 需要通过演示建立直觉

看完视频后，通常要回到文档继续核对命令、配置和风险边界。

## 如果你想知道它能做成什么样

优先进入：

- [案例展示](/showcase)

适合：

- 想看团队和个人场景是怎么落地的
- 更关心结果，而不是先看底层机制
- 需要向同事解释“这东西有什么实际价值”

## 如果你已经遇到具体问题

按问题类型选：

- 安装/启动：去 [安装与环境](/docs/setup/installation)
- 模型：去 [模型选择指南](/models)
- 配置：去 [关键配置](/configurations)
- 安全/远程访问：去 [安全实践](/security)
- 搜不到内容：去 [站内搜索](/search)

## 2026 年 3 月 24 日的中文阅读入口观察

近期公开可访问的中文教程站、社区文章和媒体内容，正在把更多注意力放到：

- 国内模型 provider
- 飞书 / 微信 / 企业协作入口
- Docker / 云部署
- 本地模型和自动化能力

这说明中文用户最常被吸引的入口，已经不再只是“它是什么”，而是“它能不能接进我现在的工作流”。  
但这也会带来一个副作用：很多人会直接跳到最热的话题页，而不是先判断自己现在真正缺哪一层信息。

所以这页更想帮你做的一件事是：

- 如果你现在还没跑通，就先别被热门话题带走
- 如果你已经明确目标，再按路径、主题或专题页进入

一个更稳的判断方式通常是：

1. **还没跑通最小链路**：先看入门页和安装页  
2. **已经跑通，想系统学习**：进学习路径  
3. **已经知道问题属于哪个主题**：进主题中心  
4. **已经知道自己在查模型、渠道、安全或二次开发**：直接进对应专题页

## 最短判断表

- 想先跑通：去 [快速入门](/docs/getting-started/getting-started)
- 想按顺序学：去 [学习路径](/paths)
- 想按主题查：去 [主题中心](/topics)
- 想先看演示：去 [视频教程](/videos)
- 想看结果：去 [案例展示](/showcase)
