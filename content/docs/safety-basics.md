# 安全配置基础

对 OpenClaw 这种可运行、可接渠道、可扩展的系统来说，安全不是上线后再补的事情，而是第一轮配置就必须考虑的部分。

## 为什么这里要先讲安全

因为一旦你开始做下面这些事情，风险面就会立刻扩大：

- 开远程访问
- 接外部消息渠道
- 暴露管理界面
- 给工具和插件更高权限

所以，中文站点在冷启动阶段就应该把“安全基础”放进核心内容，而不是藏在附录里。

## 最基础的四个原则

### 1. 先跑本地，再谈公开入口

第一次安装时，尽量先在本地验证最小链路，不要一开始就把入口暴露到公网。

### 2. 管理入口和使用入口分开看

Control UI 这类管理入口的安全要求，通常应高于普通聊天入口。不要把它们当成同一类东西处理。

### 3. 新增渠道就新增一层边界

每接一个渠道，都等于多一个入口、多一组凭证和多一层潜在错误配置。

### 4. 先最小权限，后逐步放开

无论是工具、hooks 还是远程能力，都应优先从最小范围开始，而不是一次性全部开放。

## 中文用户最容易踩的坑

### 只关注能不能跑，不关注是不是暴露过头

这会导致系统虽然“跑起来了”，但已经处在不合适的公开状态。

### 在不了解边界前就接多个外部入口

这样一旦出错，排查复杂度会迅速升高。

### 把安全理解成后期运维问题

实际上，它从第一次安装就开始了。

## 冷启动阶段的最低建议

- 先本地验证
- 先一个入口
- 先最小权限
- 先明确谁能访问管理界面
- 再决定是否开放远程访问
