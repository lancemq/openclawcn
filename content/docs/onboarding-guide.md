# Onboarding 引导流程说明

从官方文档结构看，OpenClaw 非常重视 onboarding。这个信息很关键，因为它意味着官方并不要求你第一次使用就理解全部内部结构，而是希望你先完成一轮受控的初始化。

## 为什么 onboarding 很重要

对中文用户来说，第一次接触 OpenClaw 通常会遇到三个问题：

- 不清楚先从哪里开始
- 不知道 Control UI、渠道和工具之间是什么关系
- 不确定哪些配置是现在就必须做的

onboarding 的价值，就是把这三个问题的顺序理清楚。

## 第一次配置最应该关注什么

建议把注意力放在下面几件事：

### 1. 先确认运行环境

不要急着扩展渠道和高级能力，先确认：

- 当前系统环境能正常跑起来
- 基础安装没有报错
- 最小链路已经能工作

### 2. 再确认入口

第一次使用时，最重要的是知道自己准备通过什么入口和 OpenClaw 交互：

- 是先通过 Control UI
- 还是先通过某个消息渠道
- 还是先从本地测试链路开始

### 3. 最后再扩展

完成第一轮 onboarding 后，再逐步进入：

- 更多渠道
- 更多工具
- hooks
- 安全与远程访问

## 中文用户常见误区

### 一开始就想把所有能力都接上

这会直接把第一次体验变成排错现场。更好的方法，是先跑通一条最小链路。

### 把 onboarding 当成一次性的步骤

实际上，onboarding 更像“第一轮可控配置”，它不是终点，而是后续迭代的起点。

### 只关注界面，不关注边界

OpenClaw 不是只有界面体验，入口、权限、模型、渠道和部署边界要一起看。
