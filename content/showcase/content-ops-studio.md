# 内容团队选题与发布工作台

## 背景

内容团队最容易失控的地方通常不是写作本身，而是：

- 选题散
- 素材散
- 已有内容之间没有回链
- 发稿前总要临时补标签、描述和站内链接

当团队规模不大时，这些问题尤其容易拖慢更新频率。

## 方案

OpenClaw 在这条链路里主要负责：

1. 每周整理候选选题
2. 按“教程 / 更新 / 案例”三类分流
3. 生成草稿框架与发布检查单
4. 校验是否补齐描述、标签和站内回链

## 关键做法

- 只保留少量固定内容类型，避免栏目无限增殖
- 草稿由 AI 先搭结构，最终判断由编辑完成
- 每篇内容发布前都检查是否回链到文档、路径或案例
- 周更节奏优先稳定，不追求每篇都做成长文

## 结果

- 团队能稳定执行每周更新
- 新闻和教程不再相互脱节
- 内容发布更像流水线，而不是临时冲刺
- 站内搜索和专题页能吃到更完整的内容网络

## 适合谁

- 产品中文站
- 开源项目内容站
- 小型研究/教程团队
- 需要兼顾时效和长期积累的网站
