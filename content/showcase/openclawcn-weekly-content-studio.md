# OpenClaw 中文内容站周更助手

## 背景

一个围绕 OpenClaw 的中文资料站，早期最容易遇到的问题不是“没有内容”，而是：

- 页面慢慢变多，但更新节奏不稳定
- 教程、新闻、案例之间缺少联动
- 每次发稿都要重新想选题和结构
- 内容能发布，但很难形成持续曝光

如果这类站点想提升曝光率，最先需要的往往不是更多页面，而是一条稳定的内容生产链路。

## 挑战

这类内容团队通常都很小，常见约束包括：

- 只有 1 到 2 个主要维护者
- 每周可投入的内容时间有限
- 上游信息来自官方更新、社区讨论和站内已有页面，比较分散
- 需要兼顾搜索、时效和转化，但不可能每篇都写得很重

换句话说，问题不是“不会写”，而是“没有一套轻量但稳定的周更方法”。

## 解决方案

这里使用 OpenClaw 把周更流程拆成四段：

1. 收集线索
2. 固定选题
3. 生成草稿
4. 发布前检查

整个链路不是追求全自动，而是尽量减少重复整理和结构化写作的成本。

### 流程结构

```text
官方更新 / 社区线索 / 站内页面变化
                ↓
        OpenClaw 选题整理助手
                ↓
    每周选出 教程 / 更新 / 案例 三类主题
                ↓
      生成结构化草稿与发布检查单
                ↓
           人工审校与正式发布
```

### 核心分工

#### 1. 选题整理

OpenClaw 每周先把零散输入归成三类：

- 适合做教程的问题
- 适合做更新速览的变化
- 适合做案例展示的场景

这样维护者不需要每次从头思考“这周到底写什么”。

#### 2. 草稿框架生成

每一类内容使用不同模板：

- 教程：问题、步骤、坑点、下一步
- 更新：变化、影响、建议、回链
- 案例：背景、挑战、方案、效果

OpenClaw 负责先把结构搭好，人再做判断和精修。

#### 3. 发布前检查

在正式发布前，再让 OpenClaw 检查：

- 标题是否足够明确
- 描述是否能承接搜索点击
- 是否补齐标签和更新时间
- 是否有站内回链

这一步能显著减少“文章发了，但没有形成站内流动”的问题。

## 配置重点

### 内容助手的任务定义

```md
你是中文内容运营助手。

每周目标：
1. 选出 1 个教程主题
2. 选出 1 个更新主题
3. 选出 1 个案例主题

输出要求：
- 每个主题都说明为什么值得写
- 每个主题给出建议标题和摘要
- 标出最适合回链的站内页面
```

### 发布前检查提示

```md
检查下面这篇文章是否满足发布要求：

- 标题是否清晰
- description 是否具体
- tags 是否可检索
- 是否有至少 2 个站内回链
- 是否能归入 教程 / 更新 / 案例 之一

如果有问题，请直接给修改建议。
```

## 实施效果

在这条链路下，最明显的变化不是单篇文章更长，而是周更开始稳定：

- 每周都能固定发出 3 篇不同职责的内容
- 写作时间更多花在判断和修正，而不是重复搭框架
- 新闻不再是孤立页面，会回链到教程和案例
- 案例页不再只是展示，而能反向带动教程阅读

## 为什么这个案例值得参考

这个案例的价值不在于“用了很复杂的自动化”，而在于它很适合小团队复制：

- 输入来源清楚
- 内容结构固定
- 人工仍然保留最后判断
- 整套流程不会重到难以长期维护

如果你也在做围绕产品、开源项目或知识库的中文内容站，这种方法通常比“想到什么写什么”更能稳定带来曝光。

## 适合谁直接照搬

- 需要把更新翻译成中文解读的内容运营
- 希望同时做 SEO、时效内容和案例页的站点
- 想先建立周更节奏，再逐步扩自动化的人
