# OpenClawCN 网站当前实现分析

## 1. 网站定位

OpenClawCN 当前不是传统“产品官网”，而是一个面向中文用户的内容导航站、学习入口站和更新追踪站。

它服务三类核心任务：

1. 帮新用户先判断自己该从哪条路径进入。
2. 帮已上手用户在文档、专题、实践和视频之间快速切换。
3. 帮长期使用者持续跟踪更新、找到支持入口并完成反馈。

## 2. 技术实现概览

### 2.1 技术栈

- 框架：`Nuxt 3`
- 内容系统：`@nuxt/content`
- 图片：`@nuxt/image`
- 分析：`@vercel/analytics`
- 部署：`Vercel Nitro`
- 内容源：本地 Markdown 内容库

### 2.2 当前内容规模

- 文档：63 篇
- 新闻：46 篇
- 最佳实践：46 篇
- 视频条目：49 条
- 工具专题：6 个
- 主题分类：12 个

### 2.3 路由与生成策略

当前主站点由列表页、详情页、专题页和服务页组成：

- 列表页：`/docs`、`/news`、`/best-practices`
- 详情页：`/docs/[...slug]`、`/news/[...slug]`、`/best-practices/[...slug]`
- 专题页：`/paths`、`/topics`、`/videos`、`/tools`、`/skills`、`/configurations`、`/models`、`/security`、`/ecosystem`、`/secondary-development`
- 服务页：`/community`、`/search`、`/faq`、`/feedback`

首页、频道页和专题页均已参与 prerender；同时站点已具备 `rss.xml`、`sitemap.xml`、`robots.txt` 和运行时内容清单。

## 3. 当前信息架构

### 3.1 一级结构

站点当前已经切换到 5 组导航：

1. `开始`
2. `文档`
3. `进阶`
4. `动态`
5. `支持`

这套结构不再按“功能模块”硬拆，而是按“用户此刻为什么来这里”组织。

### 3.2 五层信息模型

首页和数据层对应的是同一套五层模型：

1. 开始：学习路径、主题中心、视频教程
2. 文档：文档中心、关键配置、模型选择、安全实践
3. 进阶：最佳实践、工具系列、Skills、案例展示、二次开发专题
4. 动态：新闻动态、产品路线图、生态项目、背景故事
5. 支持：搜索、FAQ、社区、反馈、下载

它的目标是降低第一次进入时的认知噪音，让首页负责分流，频道页负责承接，详情页负责展开。

## 4. 首页现状

首页现在是“导读型首页”，而不是功能卡片堆叠页。当前主要模块包括：

- Hero 主视觉与主 CTA
- 五层信息分层卡
- 开始区：用户分流 + 精选学习路径
- 主题中心
- 文档入口
- 最佳实践
- 视频教程
- 能力扩展
- 本周推荐
- 最近更新
- 生态与路线
- 支持入口
- RSS / 邮箱订阅

最近一轮已经做过收口：

- 删除了旧的“5 分钟快速开始”独立模块
- 删除了重复的“站点方向 / 重点导览”模块
- 把首页前半段压缩成“先判断入口，再选择路径”的更短主线

## 5. 主要页面状态

### 5.1 文档中心

文档中心已具备：

- 分类筛选
- 标签筛选
- 推荐阅读顺序
- 推荐起点
- 文档详情页 TOC、前后篇、相关推荐

### 5.2 新闻动态

新闻列表页已具备：

- 分类筛选
- 月份归档
- 精选推荐
- 时间线式承接

新闻详情页承担的不只是“发布消息”，还负责把短期变化重新引回文档和实践。

### 5.3 最佳实践

最佳实践列表页已具备：

- 分类筛选
- 难度筛选
- 精选推荐
- 详情页相关实践和同路径推荐

它的定位是长期方法，而不是基础入门。

### 5.4 路径页与主题页

`/paths` 负责按阶段分流用户。

`/topics` 负责按问题域重组文档、新闻、实践和视频。

这两页共同构成了站点最核心的“内容重组能力”。

### 5.5 进阶专题

`/tools`、`/skills`、`/configurations`、`/models`、`/security` 和 `/secondary-development` 已经形成进阶内容带。

它们共同承接已完成基础部署的用户。

## 6. 运营与分发基础

站点当前已具备这些基础能力：

- 内容校验与 manifest 生成
- 订阅表单与 RSS
- 反馈表单
- Vercel Web Analytics 页面访问统计
- Vercel Deploy Hook 重建入口

最近一轮已经补上最小运营归因：

- Hero CTA 点击事件
- 首页学习路径点击事件
- RSS 打开 / 复制事件
- 邮箱订阅提交事件
- 订阅接口来源字段：`page / section / cta / entryType`

这意味着站点已经从“只有入口”升级到“开始能知道入口效果”。

## 7. 当前最需要继续优化的方向

接下来的重点不再是补页面数量，而是持续收口：

1. 继续统一频道页模板，减少每页各自生长。
2. 继续削减首页中后段重复承接。
3. 把内容元数据标准执行到全部新闻与专题内容。
4. 建立按栏目、按 CTA、按订阅入口的周度复盘节奏。
