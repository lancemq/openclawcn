# 下一阶段优化计划

## 当前判断

站点已经完成公开可访问的中文资料站形态，当前最需要优化的不是再继续平铺入口，而是把内容标准、信息结构、搜索关联和长期运营能力收紧。

## 优先级

### 1. 内容治理

目标：统一内容标准，减少重复表达和来源混乱。

本阶段重点：
- 统一 `title / description / category / tags / sourceType / updatedAt` 元数据
- 区分官方、GitHub、社区、媒体、第三方和内部资料
- 收紧衍生品、新闻、视频的收录和引用标准
- 增加内容校验，尽早发现空描述、缺少标签、来源不明确等问题

可验证结果：
- 构建前能自动检查核心元数据
- 新增内容时不再依赖人工记忆字段规范
- 内容清单能输出更完整的元信息

### 2. 信息架构收口

目标：让用户更容易理解站点的主线和次级入口。

本阶段重点：
- 强化 `路径 / 主题 / 文档` 作为第一层学习入口
- 把 `视频 / 实践 / 衍生品` 明确成补充层
- 把 `新闻 / 社区 / 搜索 / 反馈` 归入服务层
- 检查首页、导航、页脚和专题页是否在表达同一套结构

可验证结果：
- 首页、导航、页脚和专题页不再各自组织一套逻辑
- 用户更容易判断“该先看哪里”

### 3. 搜索与关联阅读增强

目标：让站点更像知识库，而不是内容集合页。

本阶段重点：
- 搜索结果支持按类型、标签、难度筛选
- 文档、视频、实践、衍生品详情页增加相关推荐
- 路径页增加更明确的进度感和回流入口
- 主题页强化跨模块内容组织

可验证结果：
- 任意一篇内容看完后，都能继续进入下一篇相关内容
- 用户不容易在细节页里走丢

### 4. 运营稳定性

目标：让站点适合长期维护，而不是只适合当前内容量。

本阶段重点：
- 死链和外链有效性检查
- 内容过期复核提醒
- SEO 和结构化数据回归检查
- 排查当前构建里偶发的 Nuxt Content prerender 查询异常

可验证结果：
- 发布前检查能覆盖更多内容问题
- 构建和预渲染稳定性提升

## 执行顺序

1. 先完成内容治理
2. 再做信息架构收口
3. 然后做搜索与关联阅读增强
4. 最后处理运营稳定性

## 当前已启动

本轮先落第 1 阶段：
- 补内容元数据标准
- 增强内容校验脚本
- 扩展发布清单里的内容元信息
