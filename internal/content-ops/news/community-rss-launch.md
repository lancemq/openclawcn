# 社区与 RSS 跟踪方式说明

如果你希望长期跟踪 OpenClaw，最实用的不是一次性读完全部资料，而是建立自己的跟踪路径。

## 推荐组合

- FAQ 负责解决高频问题
- 反馈负责补齐中文资料缺口
- GitHub 负责公开讨论与问题追踪
- RSS 负责长期跟踪版本和内容变化

## 更适合哪类用户

- 想长期关注变化的人适合 RSS
- 想快速解决问题的人先看 FAQ
- 想补充中文内容的人适合反馈
- 需要公开协作的人适合 GitHub
